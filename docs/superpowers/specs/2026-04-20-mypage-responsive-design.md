# 마이페이지 반응형 개선 설계

**날짜:** 2026-04-20
**파일:** `index.html` (단일 파일 프로젝트)

---

## 문제

1. **마이페이지** `#viewMypage` 내부 그리드가 인라인 스타일 `grid-template-columns:1fr 1fr`로 고정되어 기존 미디어쿼리가 override하지 못함 → 모바일에서 2컬럼이 화면 밖으로 삐져나옴
2. **상단 필터** pills가 모바일에서 4줄로 wrap되어 지도/목록 콘텐츠 영역을 크게 잠식함

---

## 변경 범위

### 1. 마이페이지 — 탭 구조 (모바일)

**모바일 (≤600px):** 즐겨찾기 / 설정 탭 전환

```
┌──────────────────────────────┐
│ ← 마이페이지  jyjy  [로그아웃] │
├──────────────────────────────┤
│  ⭐ 즐겨찾기  │  ⚙️ 설정     │  ← 탭 바
├──────────────────────────────┤
│ 즐겨찾기 탭:                  │
│  - 즐겨찾기 목록 (상태 배지)  │
│  - 전체 해제 버튼             │
├──────────────────────────────┤
│ 설정 탭:                      │
│  - 닉네임 (읽기전용)          │
│  - 텔레그램 ID (입력, UI전용) │
│  - 집 주소 (입력 + 저장)      │
│  - 알림 키워드 (목록 + 추가)  │
└──────────────────────────────┘
```

**데스크탑 (>600px):** 기존 2컬럼 그리드 유지
- 왼쪽 컬럼(집주소 + 알림키워드)에 텔레그램 ID 카드 추가
- 오른쪽 컬럼: 즐겨찾기 유지

### 2. 텔레그램 ID 필드

- **저장:** `localStorage` 임시 저장 (API 연동은 추후)
- **위치:** 모바일 설정 탭 + 데스크탑 왼쪽 컬럼 (닉네임 바로 아래)
- **형태:** 입력 + 저장 버튼 (집주소 카드와 동일 패턴)

### 3. 모바일 필터 — 가로 스크롤

**모바일 (≤600px):** 각 필터 행에 `overflow-x: auto` + `white-space: nowrap` + `scrollbar-width: none`

```
행1 (상태): [전체] [● 접수중] [● 안내중] [● 접수종료] [마감] →→
행2 (시간대): [전체] [평일] [주말] [주간] [야간] [새벽] →→
```

검색/구/즐겨찾기/정렬 위 행은 그대로 유지.

### 4. 인라인 스타일 → 클래스 추출

`#viewMypage` 내 `display:grid;grid-template-columns:1fr 1fr` 인라인 스타일을 `.mypage-grid` 클래스로 추출하여 미디어쿼리에서 override 가능하도록 변경.

---

## 건드리지 않는 것

- 데스크탑 topbar/필터 동작
- 기존 JS 로직 (switchTab, saveHomeAddress, addMpKeyword 등)
- API (`api/seoul.js`)
- 지도/목록 뷰

---

## 구현 방식

- `<style>` 블록 내 기존 `@media(max-width:600px)` 규칙에 추가
- 마이페이지 탭 전환: 별도 `switchMypageTab(tab)` 함수 추가
- 텔레그램 ID: `localStorage.getItem/setItem('telegramId_'+currentUser)` 패턴 사용
